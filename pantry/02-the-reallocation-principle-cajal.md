## Figure 4 — Where does a freed hour go?
Priority: Important · Type: process flowchart (decision tree)

BLOCK 1 — ILLUSTRAE PASTE BLOCK:
Draw a compact decision tree answering where a freed hour from a skip should be spent. Begin at a single entry node representing a freed hour. From it, branch to a first diamond decision testing whether fewer than about ten live conversations are going; a yes-branch routes to a terminal node for networking. The no-branch flows to a second diamond testing whether a deployed, metric-bearing portfolio project exists from the last ninety days; a no-branch routes to a terminal node for portfolio work. Only when both conditions are healthy does the final branch route to a terminal node for additional targeted applications. Lay the tree top-to-bottom with single-direction arrows, diamonds for decisions and rounded rectangles for terminal actions. Make the "additional applications" terminal visually subordinate, reached last and only after two gates, to encode that it is the lowest-priority destination.
